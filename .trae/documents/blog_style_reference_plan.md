# 博客设计参考与改版计划

## Summary

- 目标：为当前博客筛选可直接预览的设计参考，并基于用户已选择的“书卷极简”方向，形成后续可直接执行的前端改版计划。
- 方向结论：优先采用“内容优先 + 中文书卷气 + 编辑部式留白 + 克制微动效”的路线，不走强赛博或强新粗野风。
- 参考站点（可直接预览）：
  - AstroPaper Demo: https://astro-paper.pages.dev/
  - Benedict Evans: https://www.ben-evans.com/
  - Blake Crosley: https://blakecrosley.com/
  - Astro Cactus Demo: https://astro-theme-cactus.netlify.app/

## Current State Analysis

### 现有技术与页面结构

- 当前前端为 Astro 项目，主要公开页面集中在：
  - `src/layouts/BaseLayout.astro`
  - `src/pages/index.astro`
  - `src/pages/articles.html.astro`
  - `src/styles/site.css`
  - `src/components/Header.astro`
  - `src/components/Footer.astro`
  - `src/components/FloatingMenu.astro`
- 当前视觉基底已经具备可延续资产：
  - 暖色纸张背景与深浅主题切换
  - Serif 标题 + Sans UI 的中文排版基础
  - 面板化布局、圆角、毛玻璃和柔和阴影
  - 首页与文章页均是“信息块 + 阅读流”的结构

### 当前视觉问题

- 首页模块较多，信息密度和视觉重心同时偏高，导致“有设计元素，但不够高级、克制”。
- 玻璃感、圆角、阴影、按钮、动效同时出现，容易削弱“书卷极简”的气质。
- 首页 Hero、文章列表、分类卡片、留言板、友链都采用相近的面板表现，层级区分不足。
- 文章页导航面板、正文面板、大纲面板并列较强，更偏“工具型后台感”，不够像“沉浸式阅读界面”。

### 参考站点与可借鉴点

#### 1. AstroPaper

- URL: https://astro-paper.pages.dev/
- 借鉴原因：
  - 结构极克制，内容优先，首页直接服务“读什么”。
  - 信息层级清楚，适合博客而不是作品集。
  - 搜索、归档、标签等内容入口组织很干净。
- 不建议照搬点：
  - 过于朴素，若直接套用会丢掉你站点现有的个人气质。

#### 2. Benedict Evans

- URL: https://www.ben-evans.com/
- 借鉴原因：
  - 很强的“编辑部 / 专栏作者”气质，适合长文与观点型内容。
  - 首页强调标题、栏目、订阅关系与长期阅读信任感。
  - 页面把“文字本身”作为主视觉，极适合书卷极简路线。
- 不建议照搬点：
  - 目前你的首页不只是文章流，还有关于我、留言板、友链，因此不能做成纯 newsletter 站。

#### 3. Blake Crosley

- URL: https://blakecrosley.com/
- 借鉴原因：
  - 模块分区高级，卡片节奏和版面秩序感很强。
  - Hero、案例区、文章区、联系区的切换自然，适合你首页多模块结构。
  - 可借鉴它的“更像编辑页面，而不是普通卡片列表”的分组方式。
- 不建议照搬点：
  - 它偏作品集与商业展示，不能让博客首页变成个人品牌销售页。

#### 4. Astro Cactus

- URL: https://astro-theme-cactus.netlify.app/
- 借鉴原因：
  - 轻量、整洁、节制，列表结构非常适合文章与碎片内容并存。
  - 顶部导航、文章列表、短内容区的编排简单有效。
  - 适合作为“结构减法”的参考。
- 不建议照搬点：
  - 视觉辨识度偏弱，不足以支撑“更具设计感与美感”的目标。

## Proposed Changes

### 总体方案

- 采用“Benedict Evans 的内容气质 + AstroPaper 的信息组织 + Blake Crosley 的版面秩序 + 保留你现有暖纸张配色”的混合方案。
- 目标不是整体换肤，而是在现有架构上做增量升级，让博客更像“精心设计的中文个人刊物”。

### 具体文件改动计划

#### `src/styles/site.css`

- 调整全局设计 token：
  - 收窄圆角体系，减少“产品卡片感”
  - 弱化玻璃模糊与高亮描边，提升纸面与排版质感
  - 拉开标题、正文、辅助信息的字号与间距层级
  - 增强分割线、留白、段落节奏，减少依赖阴影制造层次
- 新增或重构样式模块：
  - 编辑部风格 section header
  - 文章型卡片与目录型列表的两套视觉规范
  - 首页重点文章区、栏目区、关于区的差异化版式
  - 文章页沉浸式阅读布局与更克制的侧边工具层

#### `src/layouts/BaseLayout.astro`

- 保留现有布局骨架，但收紧全局氛围层：
  - 让背景装饰更安静
  - 让页面主容器更像“刊物页”而不是“浮层容器”
- 如有必要，仅做轻量结构补位，避免破坏现有插槽体系。

#### `src/components/Header.astro`

- 计划将头部从“玻璃胶囊导航”向“更薄、更稳、更像刊物页眉”的方向微调。
- 保留现有功能，不改导航文案，只调整视觉层级、间距和状态表现。

#### `src/components/FloatingMenu.astro`

- 保留浮动菜单能力，但降低存在感。
- 使其更像阅读辅助工具，而不是主视觉元素，避免与正文争抢注意力。

#### `src/pages/index.astro`

- 首页改造重点：
  - Hero 从“强展示面板”改为“更像封面导语”
  - 最新文章提升为最核心阅读入口
  - 分类区从同质卡片改为更有秩序的栏目编排
  - 关于我 / 留言板 / 友链通过版式而不是堆叠相同卡片来区分
- 保持现有文本内容不变，只调整布局、层级、容器和视觉语言。

#### `src/pages/articles.html.astro`

- 文章页改造重点：
  - 强化正文阅读区权重，让正文成为绝对主角
  - 降低左侧文章列表和右侧大纲的视觉噪音
  - 让筛选、搜索、阅读进度、专注模式看起来更统一
  - 移动端保持现有交互能力，但视觉上更简洁、更轻
- 保留现有功能逻辑，优先做视觉与版式升级，不重写数据流。

#### 保持不改

- `api/` 下接口文件不动
- `public/admin.html` 不动
- 文章内容、按钮文案、数据库结构不动

## Assumptions & Decisions

- 已确认用户偏好风格为“书卷极简”。
- 设计目标是提升公开博客前台的质感与美感，不涉及后台和 API 重构。
- 本次执行应坚持增量修改：优先复用现有组件、脚本和页面结构。
- 视觉取向上，保留当前暖色纸张基底与中文 serif 气质，不切换到纯黑科技或高饱和撞色风。
- 功能取向上，不新增复杂交互作为主目标，重点是版式、排版、层级、留白、卡片规范与整体节奏。

## Verification Steps

- 执行 `npm run build`，确认 Astro 页面可正常构建。
- 手动检查首页：
  - Hero、最新文章、专栏、关于我、留言板、友链均正常显示
  - 视觉层级更清晰，且未修改文案
- 手动检查文章页：
  - 搜索、分类筛选、文章切换、目录生成、阅读进度、评论区、移动端底部导航均保持可用
  - 阅读主区域权重明显高于辅助面板
- 手动检查 `public/admin.html` 未受影响。
- 如需进一步确认视觉方向，可在执行阶段启动 `npm run dev` 并逐页预览首页与文章页。
