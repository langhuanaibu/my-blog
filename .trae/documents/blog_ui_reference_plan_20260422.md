# 博客 UI 参考检索计划

## Summary

- 目标：为当前博客筛选一批“可直接在线预览”的现成 UI 参考站点/主题 Demo，帮助用户选择下一版更有设计感、更美观的样式方向。
- 结论：当前博客已具备“暖纸张 + 中文阅读 + 编辑部感”的基底，不适合直接跳到强赛博或强商业落地页风格；更适合在“高级阅读感 / 极简内容导向 / 设计师式分区编排”这几个方向里选。
- 本次计划不涉及代码实现，只产出参考方案、预览网址、适配建议和后续选型路径。

## Current State Analysis

### 已确认的前台结构

- 首页主入口在 `src/pages/index.astro`
- 文章阅读页在 `src/pages/articles.html.astro`
- 全局布局在 `src/layouts/BaseLayout.astro`
- 全局样式在 `src/styles/site.css`
- 顶部导航和浮动菜单分别在 `src/components/Header.astro`、`src/components/FloatingMenu.astro`

### 当前视觉基底

- 已是偏“书卷 / 杂志 / 阅读型”的博客，不是纯开发者后台感
- 现有色彩为暖米色纸张系，标题使用中文衬线字体，正文和 UI 使用无衬线字体
- 首页结构是 `Hero + 最新文章 + 专栏 + 关于我 + 留言板 + 友链`
- 文章页结构是 `文章列表 + 正文 + TOC` 的三栏阅读布局，并有移动端底部导航/抽屉交互
- 当前项目适合参考“内容型博客 / 编辑部风 / 设计师个人站”方向，而不是完全改造成 SaaS 官网或强交互实验站

### 现有约束

- 用户要求的是先做参考调研与选型，不是立即改代码
- 按项目规则，后续若只改 UI，应避免擅自修改文章文本和展示文案
- 方案必须优先考虑 Astro 博客结构可落地性，而不是纯视觉灵感图

## Proposed Changes

### 本次输出内容

- 整理 5 个可直接预览的参考站/主题 Demo
- 每个参考对象都给出：
  - 预览网址
  - 风格关键词
  - 适合借鉴的部分
  - 不建议照搬的部分
  - 对当前博客的适配判断
- 最终把它们归纳成 3 条可选改版路线，方便后续执行时选一种

### 候选参考对象

#### 1. AstroPaper

- 预览网址：https://astro-paper.pages.dev/
- 风格关键词：极简、内容优先、轻量、开发者博客
- 适合借鉴：
  - 首页层级非常清楚，文章入口突出
  - 归档、标签、搜索等内容组织方式成熟
  - 适合作为“减少杂讯、提升阅读效率”的基准样式
- 不建议照搬：
  - 默认视觉过于朴素，直接套用会削弱你博客现在已有的气质
- 适配判断：
  - 很适合参考它的“信息组织方式”，但不适合完全照搬视觉

#### 2. Astro Cactus

- 预览网址：https://astro-theme-cactus.netlify.app/
- 风格关键词：克制、轻量、专注内容、结构简洁
- 适合借鉴：
  - 文章列表和短内容并列时的节奏感
  - 导航和内容列表的简洁布局
  - 适合作为“做减法”的参考
- 不建议照搬：
  - 视觉辨识度一般，不足以单独支撑“更有设计感”的目标
- 适配判断：
  - 适合借它的结构，不适合单独作为最终视觉方向

#### 3. Benedict Evans

- 预览网址：https://www.ben-evans.com/
- 风格关键词：专栏作者、编辑部、文字主视觉、信任感强
- 适合借鉴：
  - 让文字本身成为主视觉
  - 首页栏目关系和内容权重很明确
  - 很适合长文、观点型和持续更新型博客
- 不建议照搬：
  - 你当前首页不只有文章流，还有关于我、留言板、友链，不能改成纯 newsletter 结构
- 适配判断：
  - 非常适合参考它的“高级阅读感”和“刊物气质”

#### 4. Blake Crosley

- 预览网址：https://www.blakecrosley.com/
- 风格关键词：设计师个人站、模块秩序感、精致分区、现代感
- 适合借鉴：
  - 首页多模块之间的切换节奏
  - Hero、项目/内容区、联系区的编排方式
  - 可以帮助你把首页做得更“像经过设计的页面”，而不是单纯卡片堆叠
- 不建议照搬：
  - 它偏作品集与个人品牌展示，不适合把你的博客改成“商业作品集首页”
- 适配判断：
  - 很适合首页版式升级，特别适合你当前多区块首页

#### 5. Jessica Hische

- 预览网址：https://jessicahische.is/awesome
- 风格关键词：强品牌感、艺术化排版、视觉个性、女性化细节
- 适合借鉴：
  - 更强的视觉识别度
  - 分区之间通过排版和图文关系建立层次
  - 适合参考它的“个性化首页气质”
- 不建议照搬：
  - 它的品牌表达和插画/作品权重很高，与你的技术博客内容结构不同
- 适配判断：
  - 适合作为“提高辨识度”的灵感来源，不适合作为整体结构模板

### 归纳后的 3 条可选路线

#### 路线 A：书卷编辑部

- 核心参考：`Benedict Evans + 现有暖纸张基底`
- 效果：更像中文个人刊物，强调阅读沉浸感、标题层级和留白
- 适合你当前博客的程度：最高

#### 路线 B：极简内容型

- 核心参考：`AstroPaper + Astro Cactus`
- 效果：更干净、更理性、更偏现代开发者博客
- 适合你当前博客的程度：中高

#### 路线 C：设计师分区型

- 核心参考：`Blake Crosley + Jessica Hische`
- 效果：首页更精致、更有设计感，区块更有层次和识别度
- 适合你当前博客的程度：高，但要控制不要过度作品集化

## Assumptions & Decisions

- 用户当前核心需求是“先看参考、先选方向”，不是立即执行改版
- 参考对象优先选择真实在线站点或可直接访问的主题 Demo，而不是只有截图的灵感文章
- 参考筛选标准以“适合博客落地”优先，不以炫技和视觉复杂度为优先
- 当前最推荐的默认方向是：
  - `路线 A：书卷编辑部`
  - 如果用户想更现代、更轻，则转向 `路线 B`
  - 如果用户更想提升首页辨识度，则转向 `路线 C`

## Verification Steps

- 检查所有参考链接是否为可直接访问的真实网址
- 确认每个参考对象都能对应到当前博客的某一类页面结构，而不是纯概念风格
- 输出时明确说明：
  - 哪个方向最适合当前博客
  - 哪几个链接适合先看
  - 后续如果用户选中路线，应优先改哪些页面和组件
