## Context

四 Tab 骨架已上线（`96aed66`）。本轮继续对标 AIHOT 深化各板块。经 grill-me 访谈确认的决策：

1. **全部动态评分**：全量档保持"抓取后立即落盘"的底账语义，管线评分完成后**把事件分回填**到当日档条目上（被预筛砍掉的无分）；前端默认只显示达标条目，可一键切换"显示全部（含低分/未评分）"。
2. **精选热点榜**：近 3 天 × 「独立信源数 × 时间衰减」排序 Top 4-6，安静时自动收缩，纯前端计算。
3. **精选每日折叠**：今天默认展开，历史日只显示日期头（带条数），点击展开。
4. **精选检索**：基于 180 天搜索索引（标题+标签）全历史匹配，命中结果**页内渲染成完整卡片流**（懒加载命中日的 daily js），清空关键词回到时间轴。
5. **主题页两级聚合**：大类=手工题材域分组，小类=27 个受控 topic_tags（已在每条精选与搜索索引中），小类卡片显示 180 天条数，点进=该标签的历史精选流；现有跨天事件线保留在页顶。
6. **日报页不动**，只给优化建议（见附录）。

已核实的数据事实：
- `search_index.js` 每行 `[date, id, tier, category, title, tags("|"连接)]`，保留 180 天——检索与标签统计的现成数据源。
- 全量档 `data/all/<date>.js` 条目 `{t,u,s,c,time}`，由 `write_all_archive`（[daily_news.py:1859](E:/桌面/个人博客/news-pipeline/daily_news.py) 附近）在抓取去重后写入，**无分数**。
- 评分在事件级（`score_and_select` 后每个 event 有 `score`），事件 `ids` 映射回 items，item 有 `url`——可按 URL 回填全量档。
- `config.yaml` 的 `topic_tags` 共 27 个受控标签。

## 一、管线：评分回填全量档

文件：`news-pipeline/daily_news.py`、`news-pipeline/config.yaml`、`news-pipeline/tests/test_pipeline.py`

1. `config.yaml` 新增 `all_view_min_score: 40`（全部动态默认展示阈值，可调）。
2. `write_all_archive` payload 增加 `min_score` 字段（从 cfg 传入）。
3. 新函数 `backfill_all_scores(events, items, date_str)`：
   - 读回 `data/all/<date>.js`（缺失/损坏则跳过并记日志）。
   - 构建 `url → score` 映射：遍历 events，对 `ev["ids"]` 中每个 item 的 url（去 query，同 fetch_all 去重口径）记 `ev["score"]`。
   - 给档内匹配条目加 `score` 字段（round 到整数），重写文件。不改 manifest。
   - 在 `main()` 中 `score_and_select` 完成后调用；独立故障域 try/except。
4. 测试新增：回填后匹配条目带分、未匹配条目无 score 键、payload.min_score 存在、重复回填幂等。

## 二、前端：精选页改版（index.html · renderPicksFeed 重写）

1. **单列时间轴**："一条线"视觉——组内卡片单列排布，左侧竖线 + 圆点 + 精确时间（HH:MM，`fmtTime` 已有）。新 CSS：`.tl`（容器 border-left）、`.tl-item`（相对定位圆点+时间标签）；卡片复用现有 `card()` 不改。
2. **每日折叠**：`state.feedOpen={date:bool}`（会话级）；今天默认 true，其余 false。日期头 `feed-day` 变为可点击开合（`▸/▾` + `N 条精选`），折叠时只渲染头（条目不进 DOM，保持轻量）。
3. **当前热点榜**（时间轴上方）：
   - 候选：已加载批次中最近 3 天的 pick 条目。
   - 热度 = `max(独立来源数, 1) × 0.5^(距今小时/36)`，仅收 `sources.length>=2 || score>=90` 的候选；排序取 Top 6，不足 2 条整个榜隐藏。
   - 每行：`序号 + 标题 + N 个信源 · X 小时前`，点击→展开该天分组并滚动高亮（复用 `pendingHighlight` + 复合 ref `date:id`）。
4. **页内检索**：分类 tabs 旁加输入框（仅精选 Tab 内，非全局搜索框）。输入≥1 词：`ensureIndex` → 在 `NEWS_INDEX` 中过滤 `r[2]==="pick"` 且标题/标签命中 → 按日期收集命中 id → `loadDates(命中日)` → 渲染成"检索结果流"（按日分组、全部展开、不折叠、隐藏热点榜）；清空回时间轴。命中条目注册走 `_date` + `refOf`（沿用防碰撞机制）。支持 `tag:标签名` 形式的精确标签过滤（供主题页跳转复用）。
5. 分类 tabs 保留现状（`setCat` 已 view-aware）。

## 三、前端：全部动态评分流（renderAllFeed 增强）

1. 顶部：分类 tabs（`state.allCat`，按条目 `c` 字段过滤）+ 切换钮「显示全部（含低分/未评分）」（`state.allShowAll`）。
2. 过滤规则：当日 payload 含 `min_score` 或任一条目有 `score` → 默认只显示 `score >= min_score` 的条目，日组尾行显示「已隐藏 N 条低分/未评分 · 点击显示」；老档（整日无评分，如 07-10 补种档）整日全显并在日期头标注「未回填评分」。
3. 行渲染加分数徽标（复用 `.score` + `scoreCls`，无分不显示）。

## 四、前端：主题页两级聚合（renderTopics 改版）

1. 页顶保留现有跨天事件线三组（📌 追踪中 / 🔥 进行中 / 🗄 已归档），整体加一个可折叠标题（默认展开追踪中，进行中/归档默认现状）。
2. 下方新增「题材地图」：大类分组常量 `TOPIC_GROUPS`（前端常量，与 config 的 topic_tags 对齐）：
   - **模型与产品**：模型发布、产品发布、开源、芯片算力
   - **研究与前沿**：研究论文、技巧观点、航天、医疗健康
   - **资本与市场**：融资并购、财报、市场行情、加密货币、宏观经济、能源
   - **政策与治理**：监管政策、诉讼纠纷、安全隐私、人事变动、教育政策
   - **国际与地缘**：地缘冲突、外交、选举政治
   - **社会与生活**：灾害事故、劳动就业、高校青年、气候环境、汽车出行
3. 小类卡片：标签名 + 180 天条数（`ensureIndex` 后统计 `r[5]` 命中）+「查看 N 条精选 →」；条数为 0 的标签不显示。
4. 点小类 → `switchTab('picks')` 并触发 `tag:标签名` 检索（复用二.4 的检索成流机制）。
5. 未来管线若新增标签，前端未分组的标签自动归入「其他」组兜底。

## 五、日报页优化建议（本轮不实施，仅记录）
