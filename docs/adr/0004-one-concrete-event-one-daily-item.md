# 同一具体事件在同一日报只保留一张卡

日报生成采用两层同日事件归并：评分前基于标题、原始标题、摘要和来源复核全部事件，选出精选与次级条目后再复核读者可见集合。复核按有重叠的有界批次执行，批次间通过低频证据键召回候选；读者可见集合持续执行“归并、重新审计、重新选位”，直到没有新增合并，最终才写分数账本。

事实报道与分析报道合并为同一张卡的来源和观点，不同进展仍独立。URL 是报道文档标识，只用于召回候选，不是事件身份；同一篇综合报道可能同时支撑多个具体事件。模型审计失败时只合并共享同一原始条目的确定性重复，其余条目保留并将质量状态标记为降级，以保证日报能够继续发布而不冒险误并。

日报文件、日期清单与事件登记表作为一个可回滚的发布单元替换；任一替换失败时恢复旧版本。RSS、搜索索引和质量记录只在该发布单元成功后更新。
