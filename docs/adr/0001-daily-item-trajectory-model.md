# 日报条目采用「来龙 → 现状 → 走向」轨迹模型，并复用现有字段

日报每条新闻原本读作一个封闭的事实点，看不出它由什么发展而来、接下来取决于什么。我们决定把每条精选组织成一条轨迹：**来龙 → 现状 → 走向**，让条目读完仍然指向未来。

关键的是**不新增面向读者的内容主字段**：`context` 承载「来龙」、`watch` 承载「走向」，只改变这两个字段的语义与生成方式，并调整详情页的呈现顺序。日报卡面在「为什么重要」之后只增加一行「走向」。

## Considered Options

- **新增第五段「脉络」**：与 `context`/`watch` 语义高度重叠，会加剧已知的段间冗余，且让卡面变长。
- **把 `context` 与 `watch` 合并成一段连续叙事**：读起来最像故事，但改动面波及 `detail`、周报与 RSS，且一次性事件只剩半段，结构不稳。

## Consequences

`watch` 的读者可见标签由「后续关注」统一改为「走向」（含 RSS），但字段名与数据语义不变，因此周报等既有消费方无需迁移。
